# 06 - 實戰與進階

## Docker Security 基礎

### 以非 root 使用者執行

```dockerfile
# ❌ 預設以 root 執行
FROM node:20-alpine
COPY . /app
CMD ["node", "server.js"]

# ✅ 使用非 root 使用者
FROM node:20-alpine
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
WORKDIR /app
COPY --chown=appuser:appgroup . .
USER appuser
CMD ["node", "server.js"]
```

### Image 安全掃描

```bash
# 使用 Docker Scout 掃描漏洞
docker scout cves my-image

# 使用 Trivy（開源工具）
docker run --rm \
  -v /var/run/docker.sock:/var/run/docker.sock \
  aquasec/trivy image my-image
```

### 安全最佳實踐

| 實踐 | 說明 |
|------|------|
| 使用特定版本的 tag | 避免 `latest`，使用 `nginx:1.25.3` |
| 非 root 使用者 | 不要以 root 身份執行 |
| 最小化 base image | 優先使用 `alpine` 或 `distroless` |
| 不要把 secret 放進 Image | 用環境變數或 Docker secrets |
| 定期更新 base image | 修補安全漏洞 |
| 使用 `.dockerignore` | 避免把 `.env`、`.git` 放進 Image |

## Resource 限制

```bash
# 限制 CPU 和記憶體
docker run -d \
  --name limited-app \
  --cpus="1.5" \
  --memory="512m" \
  --memory-swap="1g" \
  nginx

# 查看資源使用狀況
docker stats limited-app
```

在 Compose 中：

```yaml
services:
  app:
    image: my-app
    deploy:
      resources:
        limits:
          cpus: '1.5'
          memory: 512M
        reservations:
          cpus: '0.5'
          memory: 256M
```

## Docker Registry

### Docker Hub

```bash
# 登入
docker login

# Tag image
docker tag my-app username/my-app:v1.0

# Push
docker push username/my-app:v1.0

# Pull
docker pull username/my-app:v1.0
```

### 私有 Registry

```yaml
# 用 Docker 自建 registry
services:
  registry:
    image: registry:2
    ports:
      - "5000:5000"
    volumes:
      - registry-data:/var/lib/registry

volumes:
  registry-data:
```

```bash
# Push 到私有 registry
docker tag my-app localhost:5000/my-app:v1.0
docker push localhost:5000/my-app:v1.0
```

## CI/CD 整合：GitHub Actions

```yaml
# .github/workflows/docker.yml
name: Build and Push Docker Image

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ secrets.DOCKERHUB_USERNAME }}/my-app:latest
            ${{ secrets.DOCKERHUB_USERNAME }}/my-app:${{ github.sha }}
```

## Container Orchestration 概念

當你需要管理大量 Container、自動擴展、故障恢復時，就需要 orchestration 工具。

### Docker Swarm vs Kubernetes

| 比較 | Docker Swarm | Kubernetes (K8s) |
|------|-------------|-------------------|
| 學習難度 | 低 | 高 |
| 設定複雜度 | 簡單 | 複雜 |
| 功能豐富度 | 基本 | 非常完整 |
| 社群生態 | 較小 | 非常龐大 |
| 適用規模 | 小到中型 | 任何規模 |
| 市場採用 | 較少 | 業界主流 |

> **初學者建議**：先把 Docker 和 Compose 學好，之後再學 Kubernetes。

## 實戰專案 A：WordPress + MySQL + Nginx Reverse Proxy

### 目錄結構

```
wordpress-project/
├── docker-compose.yml
├── .env
└── nginx/
    └── default.conf
```

### .env

```env
MYSQL_ROOT_PASSWORD=rootpass123
MYSQL_DATABASE=wordpress
MYSQL_USER=wp_user
MYSQL_PASSWORD=wp_secret
```

### nginx/default.conf

```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        proxy_pass http://wordpress:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### docker-compose.yml

```yaml
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
    depends_on:
      - wordpress
    restart: unless-stopped

  wordpress:
    image: wordpress:php8.3-fpm-alpine
    environment:
      WORDPRESS_DB_HOST: db
      WORDPRESS_DB_USER: ${MYSQL_USER}
      WORDPRESS_DB_PASSWORD: ${MYSQL_PASSWORD}
      WORDPRESS_DB_NAME: ${MYSQL_DATABASE}
    volumes:
      - wp-files:/var/www/html
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - db-data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 3
    restart: unless-stopped

volumes:
  wp-files:
  db-data:
```

```bash
# 啟動
docker compose up -d

# 查看狀態
docker compose ps

# 查看 log
docker compose logs -f

# 停止
docker compose down
```

## 實戰專案 B：開發環境一鍵啟動

以一個典型的 full-stack 專案為例：

```yaml
# docker-compose.dev.yml
services:
  # PostgreSQL 資料庫
  db:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: devdb
      POSTGRES_USER: dev
      POSTGRES_PASSWORD: devpass
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./scripts/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U dev"]
      interval: 5s
      timeout: 5s
      retries: 5

  # Redis 快取
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  # MailHog（本地信件測試）
  mailhog:
    image: mailhog/mailhog
    ports:
      - "1025:1025"    # SMTP
      - "8025:8025"    # Web UI

  # Adminer（資料庫管理 Web UI）
  adminer:
    image: adminer
    ports:
      - "8081:8080"
    depends_on:
      - db

volumes:
  pgdata:
```

```bash
# 啟動開發環境
docker compose -f docker-compose.dev.yml up -d

# 你的應用程式（在 Host 上開發）可以連接：
# DB:      localhost:5432
# Redis:   localhost:6379
# Mail:    localhost:1025 (SMTP) / localhost:8025 (Web UI)
# Adminer: localhost:8081
```

## Docker 常見問題排解

### Container 啟動後馬上停止

```bash
# 查看 log
docker logs <container>

# 常見原因：
# 1. 主行程結束了（CMD 的指令執行完就退出）
# 2. 設定錯誤（如資料庫密碼不對）
# 3. port 被佔用
```

### Port 已被佔用

```bash
# 查看哪個行程佔用了 port
# Windows
netstat -ano | findstr :8080

# 解法：改用其他 port 或停止佔用的行程
```

### Volume 權限問題

```bash
# 查看 Container 內的檔案權限
docker exec <container> ls -la /app

# 解法：在 Dockerfile 中設定正確的權限
RUN chown -R appuser:appgroup /app
```

### Image build 很慢

```bash
# 1. 確認 .dockerignore 有排除 node_modules 等大目錄
# 2. 利用 layer cache（把不常變動的指令放前面）
# 3. 使用 multi-stage build
```

## 面試常見問題總整理

| 問題 | 關鍵概念 |
|------|---------|
| Container vs VM？ | 共享 kernel vs 獨立 OS |
| Image vs Container？ | 模板 vs 執行實例 |
| CMD vs ENTRYPOINT？ | 可覆蓋 vs 不易覆蓋 |
| COPY vs ADD？ | 單純複製 vs 額外功能 |
| Volume vs Bind Mount？ | Docker 管理 vs 使用者指定路徑 |
| 預設 bridge vs 自訂 bridge？ | 無 DNS vs 有 DNS 解析 |
| depends_on 能保證什麼？ | 只保證啟動順序，不保證 ready |
| 如何減少 Image 大小？ | alpine + multi-stage + 合併 layer |
| Docker 安全最佳實踐？ | 非 root + 特定 tag + 掃描漏洞 |
| Swarm vs Kubernetes？ | 簡單 vs 功能完整 |

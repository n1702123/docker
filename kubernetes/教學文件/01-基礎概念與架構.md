# 01 - Kubernetes 基礎概念與架構

## 什麼是 Kubernetes？

Kubernetes（簡稱 **K8s**）是一個 **容器編排平台（container orchestration platform）**，讓你能自動化地部署、擴展、管理大量的容器。

簡單來說：
> Docker 讓你跑一個容器，Kubernetes 讓你管理**成千上萬個**容器。

---

## 為什麼需要 Kubernetes？

### Docker 單機的限制

假設你的服務跑在一台機器上：

```
問題一：流量突然暴增怎麼辦？
  → 手動再開幾個 Container？要改 config？要通知 load balancer？

問題二：Container 掛掉怎麼辦？
  → 手動重啟？幾秒後才發現？中間服務中斷？

問題三：要更新版本怎麼辦？
  → 全部停掉再重起？服務中斷？要怎麼回滾？

問題四：跨多台機器怎麼辦？
  → 每台都要手動操作？
```

### Kubernetes 的解決方案

```
K8s 幫你自動處理：

  自動擴縮容   → 流量大時自動加 Container，流量小時自動縮減
  自動修復     → Container 掛掉自動重啟，節點掛掉自動遷移
  滾動更新     → 不停機更新，隨時可回滾
  跨機器管理   → 多台機器統一管理，像用一台一樣
  服務發現     → Container 之間自動找到對方
```

## Kubernetes 架構

K8s 叢集（Cluster）由兩種節點（Node）組成：

```
┌─────────────────────────────────────────────────────┐
│                   K8s Cluster                       │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │            Control Plane（Master Node）      │   │
│  │  ┌──────────┐ ┌───────────┐ ┌───────────┐  │   │
│  │  │API Server│ │ Scheduler │ │Controller │  │   │
│  │  │          │ │           │ │ Manager   │  │   │
│  │  └──────────┘ └───────────┘ └───────────┘  │   │
│  │               ┌───────────┐                 │   │
│  │               │   etcd    │                 │   │
│  │               └───────────┘                 │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  ┌──────────────┐  ┌──────────────┐                │
│  │ Worker Node 1│  │ Worker Node 2│                │
│  │ ┌──────────┐ │  │ ┌──────────┐ │                │
│  │ │ kubelet  │ │  │ │ kubelet  │ │                │
│  │ │kube-proxy│ │  │ │kube-proxy│ │                │
│  │ │ Pod Pod  │ │  │ │ Pod Pod  │ │                │
│  │ └──────────┘ │  │ └──────────┘ │                │
│  └──────────────┘  └──────────────┘                │
└─────────────────────────────────────────────────────┘
```

### Control Plane 組件（大腦）

| 組件 | 說明 |
|------|------|
| **API Server** | 所有操作的入口，kubectl 指令就是在跟它溝通 |
| **Scheduler** | 決定 Pod 要排程到哪個 Worker Node 上跑 |
| **Controller Manager** | 持續監控叢集狀態，確保實際狀態符合期望狀態 |
| **etcd** | 分散式 key-value 資料庫，儲存所有叢集資料 |

### Worker Node 組件（手腳）

| 組件 | 說明 |
|------|------|
| **Kubelet** | 每個 Node 上的代理人，負責實際管理 Pod |
| **Kube-proxy** | 管理 Node 上的網路規則，實現 Service 的流量轉發 |
| **Container Runtime** | 實際跑容器的引擎（通常是 containerd） |

## 核心概念

### Pod：K8s 最小部署單元

Pod 是 K8s 的最小部署單元，**一個 Pod 包含一個或多個 Container**。

```
┌───────────────────────────┐
│          Pod              │
│  ┌─────────┐ ┌─────────┐  │
│  │Container│ │Sidecar  │  │
│  │  (主要) │ │Container│  │  ← 通常一個 Pod 一個 Container
│  └─────────┘ └─────────┘  │
│  共享 Network namespace    │
│  共享 Storage volumes      │
└───────────────────────────┘
```

Pod 的特性：
- Pod 內的 Container 共享同一個 IP 和 Port space
- Pod 是暫時的（ephemeral），掛掉就掛掉，不會自動重建
- 通常不直接建立 Pod，而是透過 Deployment 管理

### Namespace：資源隔離

Namespace 用來把叢集資源分隔到不同的虛擬空間：

```
Cluster
├── namespace: default        ← 預設 namespace
├── namespace: production     ← 正式環境
├── namespace: staging        ← 測試環境
└── namespace: kube-system    ← K8s 系統組件（不要亂動）
```

### Label 與 Selector：資源標記

Label 是 key-value 的標籤，用來標記資源；Selector 用來選取有特定 Label 的資源：

```yaml
# 幫 Pod 打標籤
metadata:
  labels:
    app: web
    env: production
    version: v1.2

# 用 Selector 選取
selector:
  matchLabels:
    app: web
    env: production
```

## 安裝本地環境

### 方式一：Docker Desktop（最簡單）

1. 開啟 Docker Desktop
2. 進入 Settings → Kubernetes
3. 勾選 "Enable Kubernetes"
4. 點擊 "Apply & Restart"

### 方式二：Minikube（推薦學習用）

```bash
# Windows（使用 winget）
winget install Kubernetes.minikube

# 啟動 Minikube
minikube start

# 查看狀態
minikube status

# 開啟 Dashboard（可視化介面）
minikube dashboard
```

### 安裝 kubectl

```bash
# Windows（使用 winget）
winget install Kubernetes.kubectl

# 確認安裝
kubectl version --client

# 查看叢集資訊
kubectl cluster-info
```

## 基礎 kubectl 指令實作

### 1. 查看叢集狀態

```bash
# 查看叢集資訊
kubectl cluster-info

# 查看所有 Node
kubectl get nodes

# 查看 Node 詳細資訊
kubectl describe node <node-name>
```

輸出範例：
```
NAME       STATUS   ROLES           AGE   VERSION
minikube   Ready    control-plane   5m    v1.28.0
```

### 2. 建立第一個 Pod

```bash
# 直接用指令建立 Pod（快速測試用）
kubectl run my-pod --image=nginx

# 查看 Pod 狀態
kubectl get pods

# 查看 Pod 詳細資訊（排查問題很好用）
kubectl describe pod my-pod

# 查看 Pod 的 log
kubectl logs my-pod

# 進入 Pod 內部
kubectl exec -it my-pod -- bash
```

### 3. 用 YAML 建立 Pod（正式做法）

建立檔案 `my-pod.yaml`：

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
  labels:
    app: web
spec:
  containers:
  - name: nginx
    image: nginx:latest
    ports:
    - containerPort: 80
```

套用這個設定：

```bash
# 建立資源
kubectl apply -f my-pod.yaml

# 查看結果
kubectl get pods

# 刪除資源
kubectl delete -f my-pod.yaml
# 或
kubectl delete pod my-pod
```

### 4. 使用 Namespace

```bash
# 查看所有 Namespace
kubectl get namespaces

# 在特定 Namespace 建立資源
kubectl run my-pod --image=nginx -n staging

# 查看特定 Namespace 的 Pod
kubectl get pods -n staging

# 查看所有 Namespace 的 Pod
kubectl get pods -A
```

## YAML 結構說明

每個 K8s 資源的 YAML 都有四個必要欄位：

```yaml
apiVersion: v1          # API 版本（不同資源類型不同）
kind: Pod               # 資源類型
metadata:               # 元資料（名稱、標籤等）
  name: my-pod
  namespace: default
  labels:
    app: web
spec:                   # 資源的期望狀態（最重要的部分）
  containers:
  - name: my-container
    image: nginx
```

常見 apiVersion 對照：

| 資源類型 | apiVersion |
|---------|------------|
| Pod | `v1` |
| Service | `v1` |
| Deployment | `apps/v1` |
| ConfigMap | `v1` |
| Secret | `v1` |
| Ingress | `networking.k8s.io/v1` |

## 常用指令速查表

| 指令 | 說明 |
|------|------|

## 練習題

1. 確認你的本地 K8s 叢集正常運作，用 `kubectl get nodes` 確認 Node 狀態是 `Ready`
2. 建立一個名為 `hello-k8s` 的 Pod，使用 `nginx:alpine` image
3. 用 `kubectl describe` 查看這個 Pod 的詳細資訊，找到它被排程到哪個 Node
4. 用 `kubectl logs` 查看 Pod 的 log
5. 進入 Pod 內部，執行 `nginx -v` 查看 Nginx 版本
6. 建立一個名為 `dev` 的 Namespace，在裡面建立另一個 Pod

<details>

```bash
# 練習 1
kubectl get nodes

# 練習 2
kubectl run hello-k8s --image=nginx:alpine

# 練習 3

# 練習 4

# 練習 5
kubectl exec -it hello-k8s -- nginx -v

# 練習 6
kubectl create namespace dev
kubectl run dev-pod --image=nginx:alpine -n dev
kubectl get pods -n dev
```

## 面試重點整理

**Q：Kubernetes 解決了什麼問題？**
> 解決了大規模容器的自動化部署、擴縮容、自我修復、滾動更新等維運問題。當服務規模超過幾台機器，手動管理 Container 變得不可行，K8s 提供了一套標準化的管理機制。

**Q：K8s 的架構組成是什麼？**
> 分為 Control Plane 和 Worker Node。Control Plane 包含 API Server（入口）、Scheduler（排程）、Controller Manager（控制迴圈）、etcd（資料庫）。Worker Node 包含 Kubelet（執行 Pod）、Kube-proxy（網路規則）、Container Runtime（跑容器）。

**Q：Pod 是什麼？**
> Pod 是 K8s 最小的部署單元，包含一個或多個 Container，共享同一個 IP 和存儲。Pod 是暫時性的，通常不直接建立，而是透過 Deployment 等控制器來管理。

**Q：Namespace 的用途是什麼？**
> 用來在同一個叢集中做資源隔離，常用來區分不同環境（dev/staging/production）或不同團隊的資源。資源的名稱在同一個 Namespace 中必須唯一，但不同 Namespace 可以重複。

# Docker 學習計畫

> 針對初學者設計，涵蓋工作實務、面試準備、個人專案部署

---

## 階段一：基礎概念（第 1-2 週）

### 目標
- 理解什麼是 Container（容器）與為什麼需要它
- 能安裝並執行第一個 Container

### 學習內容
| 主題 | 說明 |
|------|------|
| 虛擬化 vs 容器化 | VM 與 Container 的差異 |
| Docker 架構 | Docker Engine、Docker Daemon、Docker CLI |
| Image 與 Container | Image 是模板，Container 是執行中的實例 |
| Docker Hub | 公開的 Image Registry |

### 實作練習
- [ ] 安裝 Docker Desktop（Windows）
- [ ] 執行 `docker run hello-world`
- [ ] 執行 `docker run -it ubuntu bash` 進入容器內部
- [ ] 用 `docker ps`、`docker images` 查看狀態
- [ ] 從 Docker Hub 拉取 nginx image 並啟動

### 面試重點
- Container 與 VM 的差異是什麼？
- Docker image 和 container 的關係？
- Docker 的架構組成？

---

## 階段二：Image 管理與 Dockerfile（第 3-4 週）

### 目標
- 學會建立自己的 Docker Image
- 理解 Dockerfile 的寫法與最佳實踐

### 學習內容
| 主題 | 說明 |
|------|------|
| Dockerfile 指令 | FROM、RUN、COPY、ADD、CMD、ENTRYPOINT、EXPOSE、ENV、WORKDIR |
| Build context | Docker build 時的檔案傳送機制 |
| Layer 與 Cache | Image 的分層結構與快取機制 |
| .dockerignore | 排除不必要的檔案 |
| Multi-stage build | 減少最終 Image 大小的技巧 |
| Image tagging | Tag 的命名慣例與版本管理 |

### 實作練習
- [ ] 撰寫第一個 Dockerfile（以簡單的 Node.js 或 Python app 為例）
- [ ] 使用 `docker build` 建立 Image
- [ ] 比較 CMD 與 ENTRYPOINT 的差異
- [ ] 實作 multi-stage build
- [ ] 使用 `.dockerignore` 優化 build context
- [ ] 推送 Image 到 Docker Hub

### 面試重點
- CMD 與 ENTRYPOINT 的差異？
- 如何減少 Docker image 的大小？
- Docker build cache 是怎麼運作的？
- COPY 與 ADD 的差異？

---

## 階段三：Container 操作與資料管理（第 5-6 週）

### 目標
- 熟練操作 Container 的生命週期
- 理解資料持久化的方式

### 學習內容
| 主題 | 說明 |
|------|------|
| Container 生命週期 | create → start → stop → restart → rm |
| 環境變數 | `-e` flag 與 env file |
| Port mapping | `-p` host:container 的對應 |
| Volume | Named volume 與 bind mount |
| tmpfs mount | 暫存於記憶體的資料 |
| Log 管理 | `docker logs` 與 log driver |

### 實作練習
- [ ] 用不同方式傳遞環境變數給 Container
- [ ] 設定 port mapping 讓外部存取 Container 內的服務
- [ ] 建立 named volume 保存資料庫資料
- [ ] 使用 bind mount 開發時同步程式碼
- [ ] 練習 `docker exec`、`docker logs`、`docker inspect`
- [ ] 啟動 MySQL container 並用 volume 持久化資料

### 面試重點
- Volume 和 bind mount 的差異？
- Container 停止後資料會不見嗎？
- 如何查看 Container 的 log？

---

## 階段四：Docker Network（第 7-8 週）

### 目標
- 理解 Docker 的網路模型
- 能讓多個 Container 互相溝通

### 學習內容
| 主題 | 說明 |
|------|------|
| Network driver | bridge、host、none、overlay |
| Bridge network | 預設網路與自訂網路的差異 |
| DNS resolution | Container 名稱解析 |
| 跨 Container 通訊 | 同一網路內的 Container 如何溝通 |

### 實作練習
- [ ] 建立自訂 bridge network
- [ ] 讓兩個 Container（如 app + db）透過自訂網路通訊
- [ ] 用 `docker network inspect` 查看網路設定
- [ ] 實驗不同 network driver 的效果

### 面試重點
- Docker 預設的 network driver 是什麼？
- 自訂 bridge network 與預設 bridge 的差異？
- Container 之間如何用名稱互相找到對方？

---

## 階段五：Docker Compose（第 9-11 週）

### 目標
- 用 Docker Compose 管理多容器應用
- 能撰寫完整的 `docker-compose.yml`

### 學習內容
| 主題 | 說明 |
|------|------|
| Compose 檔案格式 | services、networks、volumes 的定義 |
| 常用指令 | `docker compose up/down/ps/logs/exec` |
| 依賴管理 | `depends_on` 與 healthcheck |
| 環境變數 | `.env` 檔案與變數替換 |
| Build 整合 | 在 Compose 中 build image |
| Profile | 用 profile 區分不同環境 |

### 實作練習
- [ ] 撰寫包含 web app + database 的 `docker-compose.yml`
- [ ] 使用 `.env` 管理環境變數
- [ ] 設定 volume 讓資料庫資料持久化
- [ ] 設定 healthcheck 確保服務啟動順序
- [ ] 建立一個完整的三層架構（frontend + backend + database）
- [ ] 使用 `docker compose watch` 開發時自動重建

### 面試重點
- Docker Compose 的用途是什麼？
- `depends_on` 能保證服務啟動順序嗎？
- 如何在 Compose 中管理環境變數？

## 階段六：實戰與進階（第 12-14 週）

### 目標
- 整合所學，完成實際專案
- 了解生產環境的注意事項

### 學習內容
| 主題 | 說明 |
|------|------|
| Security 基礎 | 非 root 使用者、image 掃描 |
| Resource 限制 | CPU、Memory 限制設定 |
| Docker Registry | 私有 Registry 的建置 |
| CI/CD 整合 | 在 GitHub Actions 中使用 Docker |
| Container Orchestration 概念 | Docker Swarm / Kubernetes 簡介 |

### 實作專案
- [ ] **專案 A**：個人部落格系統（WordPress + MySQL + Nginx reverse proxy）
- [ ] **專案 B**：開發環境一鍵啟動（你工作中用到的技術棧）
- [ ] **專案 C**：用 GitHub Actions 自動 build 並 push image

### 面試重點
- 如何確保 Docker 的安全性？
- Docker Swarm 與 Kubernetes 的差異？
- 如何在 CI/CD 中使用 Docker？
